## 从零开始驱动BLDC

### 控制MOS管通断驱动直流电机

#### 直流有刷电机

直流有刷电机最直观。转子上有线圈，定子提供磁场，电刷和换向器负责在合适的时刻切换电流方向，这样转子就会一直朝同一个方向转。

它的优点是好驱动，给电就能转；缺点也很明显，电刷和换向器会磨损，转速高了以后噪声、火花和维护问题都会冒出来。

对这种电机，最常见的控制方法就是 PWM 调速。PWM 不是真的把电源电压变小，而是在一个固定周期里快速开关，让电机看到一个"平均电压"。

- 占空比越大，平均电压越高，转速通常也越高。
- PWM 频率会影响电流纹波、开关损耗和可闻噪声。
- 中心对齐 PWM 往往比边沿对齐更平滑一些，高频谐波也更少。

如果想让电机反转，就要把绕组两端的电流方向反过来。落到硬件上，就是控制 H 桥里上下 MOS 管的导通组合。控制器算出来的占空比，最后也都是通过 [MOS 常见电路](MOS.md) 里的高低边驱动和功率开关级送到电机绕组上。

#### 直流无刷电机（BLDC）

BLDC 可以看成"把有刷电机的机械换向器拿掉，再用电子开关来完成换向"。线圈放到定子上，永磁体放到转子上，控制器根据转子位置去切换各相电流，于是转子就会跟着旋转磁场跑。

这里常见的问题是：为什么大多数电机都是三相，而不是两相、四相或者更多相？

一个工程上够用的回答是：

- 两相也能转，但合成磁场脉动更明显，转矩波动通常更大。
- 三相刚好在结构复杂度、控制难度、成本和输出平滑度之间取了一个很实用的平衡。
- 更多相数当然也可以，比如五相、六相，但驱动电路和控制算法都会更复杂，常见消费级和工业伺服里没必要默认上这么高。

所以三相并不是唯一答案，而是最常见、最划算的答案。

如果先不谈 FOC，只看最基础的六步换向，那思路很简单：每次让两相导通、一相悬空，随着转子转动按顺序切换通电相别，定子磁场就会一格一格地转起来。

继续往上走，就会从六步换向进入三相正弦波控制：

- 三相参考波彼此相差 `120°`。
- 三相电流或电压矢量叠加后，合成磁场转得更连续。
- 在理想平衡三相系统里，三相瞬时和为 `0`，分析和控制都会方便很多。

#### 控制矢量合成

从控制角度看，我们真正想控制的不是某一相单独的电压，而是三相一起合成出来的空间电压矢量。

- `SPWM` 可以理解成：先给出三相正弦参考波，再和三角载波比较，最后得到三路 PWM 占空比。
- 这样做的结果，是让逆变器输出的基波尽量接近目标正弦波。
- 但电机真正"感受到"的是三相共同合成后的磁场方向和幅值。

所以后面学 FOC、SVPWM 时，脑子里最好一直盯着"矢量"这个对象，而不是把三相完全拆开来看。

### FOC的动机

磁场定向控制（Field-Oriented Control，FOC）的核心想法是：不要直接在三相坐标里硬控，而是换一个跟着转子一起转的坐标系，把电流拆成"产生磁链的部分"和"产生转矩的部分"，这样控制会清楚很多。

对于永磁同步电机或常见的 BLDC 正弦波控制场景，通常希望定子磁场尽量与转子磁场保持正交。直观地说，推着它转比顶着它转更有效，此时单位电流更容易换来转矩。

定子电流可以解耦成两部分：

- 励磁分量 `$i_d$`
- 转矩分量 `$i_q$`

**按转子磁场定向**，就是让转子磁链旋转矢量始终落在 `dq` 坐标系的 `d` 轴上，`q` 轴与它正交。这样一来：

- `$i_d$` 主要影响磁链。
- `$i_q$` 主要影响电磁转矩。

在表贴式永磁电机里，常见控制目标是让 `$i_d = 0$`，把电流尽量都用在转矩上。于是控制器会更像在控一台"直流电机"：想要多大转矩，就去调 `$i_q$`。

#### SVPWM

SVPWM（空间矢量 PWM）是把三相逆变器看成 8 个开关状态，对应 6 个有效电压矢量和 2 个零矢量。控制器在每个 PWM 周期里，选出当前扇区相邻的两个有效矢量，再配上零矢量，用时间加权的方法去逼近目标电压矢量。

相比 SPWM，它最常被提到的优势有两个：

- 直流母线电压利用率更高。
- 输出线电压的失真更小，电流波形通常也更好看。

如果从实现角度看，很多代码不会显式写出"扇区 + 作用时间"那套推导，而是走一个更容易下手的等价写法：在三相参考电压上叠加一个公共偏置，也就是零序分量，把三相一起往上或往下平移，最后让三路占空比都落进母线电压范围内。

常见写法类似这样：

`u_offset = (max(u_a, u_b, u_c) + min(u_a, u_b, u_c)) / 2`

然后再做：

`u_a' = u_a - u_offset`

`u_b' = u_b - u_offset`

`u_c' = u_c - u_offset`

这和很多资料里提到的 `mid(u_a, u_b, u_c)`、零序注入，本质上是在说同一类事：不改变线电压目标，只调整三相相对母线中点的位置。

### FOC控制

FOC 最终还是要回到一个很实际的问题：给逆变器什么三相电压，才能让电机输出目标转矩，并且系统稳定、损耗可控、响应也足够快。

完整链路通常是：

1. 采样三相电流和转子位置。
2. 用克拉克变换把三相量投到静止的 `alpha-beta` 平面。
3. 用帕克变换把它再转到跟着转子转的 `dq` 坐标系。
4. 在 `dq` 坐标系里做电流环控制，得到目标 `$u_d$`、`$u_q$`。
5. 逆帕克、逆克拉克变换回三相电压。
6. 用 SVPWM 或 SPWM 生成 PWM 占空比，去驱动 MOS 管。

#### 克拉克变换

平衡三相系统里，三相电流满足：

`i_a + i_b + i_c = 0`

这说明三相电流并不是三个完全独立的变量，实际上只要知道其中两个，第三个就能算出来。所以可以把问题投影到二维平面里处理，这就是克拉克变换的意义。

克拉克变换做的就是：

`i_a, i_b, i_c -> i_alpha, i_beta`

这里的 `$i_\alpha$`、`$i_\beta$` 固定在定子坐标系上，可以把它理解成"静止平面里的直角坐标"。

常见的恒幅值克拉克变换写法是：

`i_alpha = i_a`

`i_beta = (i_a + 2i_b) / sqrt(3)`

如果三相平衡，也可以先用 `i_c = -i_a - i_b` 消掉一相，再写成等价形式。

#### 恒幅与恒功率克拉克变换

这两个名字很像，但关注点不一样：

- 恒幅值变换：尽量让变换前后矢量幅值对应关系直观，工程实现里很常见。
- 恒功率变换：多乘一个缩放系数，让变换前后瞬时功率表达式保持一致，理论分析和一些教材里更常见。

常见的恒功率形式会写成：

`i_alpha = sqrt(2/3) * (i_a - i_b / 2 - i_c / 2)`

`i_beta = sqrt(2/3) * (sqrt(3) / 2) * (i_b - i_c)`

该怎么选，主要看你后面整套公式采用的是哪一种标定。最怕的不是选错，而是前面用恒幅值、后面 PI 参数和功率公式却按恒功率去算，结果所有量纲都对不上。

#### 帕克变换

帕克变换进一步把定子坐标系下的 $i_\alpha$、$i_\beta$ 旋转到转子坐标系：

`i_alpha, i_beta -> i_d, i_q`

这里的 $i_d$、$i_q$ 与转子磁场固连。

电角度 $\theta$ 可以理解为两个坐标系的夹角，也就是静止坐标系转到转子坐标系所需的角度。这个角度通常来自磁编码器、霍尔传感器、观测器，或者它们的组合。

公式就是一个标准旋转矩阵：

`i_d = i_alpha * cos(theta) + i_beta * sin(theta)`

`i_q = -i_alpha * sin(theta) + i_beta * cos(theta)`

逆变换则是：

`i_alpha = i_d * cos(theta) - i_q * sin(theta)`

`i_beta = i_d * sin(theta) + i_q * cos(theta)`

这一步做完之后，交流量就被"搬"到了一个旋转坐标系里。原本不断变化的三相正弦电流，在稳态下会变成更接近直流量的 `$i_d$`、`$i_q$`，这也是 PI 电流环能好用的原因。

#### 结论

走到这里，控制思路就顺了很多。

- `$i_q$` 主要对应目标转矩，所以可以理解成"所需转矩 -> 目标 `$i_q$`"。
- 为了减少无效励磁，很多场景下直接令 `$i_d = 0$`。
- 电流环的任务，是把实际 `$i_d$`、`$i_q$` 拉到目标值附近。

整体链路可以写成：

`i_q, i_d, theta -> i_alpha, i_beta -> i_a, i_b, i_c -> u_a, u_b, u_c -> PWM占空比`

如果再把闭环补完整，前面通常还有一层：

`位置/速度目标 -> 目标转矩 -> 目标 i_q`

##### 速度环

开环速度控制最简单，本质上就是：

`目标转速 -> 目标电角速度 -> 目标电压`

控制器不看电流反馈，也不真正比较当前转速误差，而是按设定值直接推着磁场转。这种方式容易上手，适合初期点亮电机、验证编码器方向、检查相序是否正确。

但它的问题也很直接：

- 负载一变，转速就会漂。
- 低速性能通常不好。
- 容易丢步或者拉不动重载。

#### PID闭环

如果对比例、积分、微分分别在解决什么问题还不熟，先看 [了解PID控制](../Miscellaneous/PID.md)，再回来看位置环、速度环和电流环会顺很多。

##### 位置环

加入磁编码器后，就能直接拿到转子位置。位置环比较"目标角度"和"当前角度"，输出一般不是直接输出电压，而是输出一个速度目标或者转矩目标，交给下一层环去处理。

##### 位置-速度环

位置环外面管"转到哪"，速度环里面管"转多快"。这样系统会比单独做位置环稳定得多，也更容易调。

- 可以通过定时采样位置差分来估算转速。
- 也可以结合编码器计数器直接得到速度。
- 速度环的输出通常会变成目标 `$i_q$`，因为 `$i_q$` 最接近"力矩指令"。

##### 位置-速度-电流环

只做位置环和速度环还不够，因为电机绕组有电感，电流不可能一步到位。你以为自己给了足够的电压，实际上电流还没跟上，转矩也就没真正到位。

所以要再加最里面的电流环，用电流传感器测量实际相电流，把它变换到 `$dq$` 坐标系里，分别控制 `$i_d$` 和 `$i_q$`。这样控制到的就不只是"电压大概够"，而是"电流确实到了"。

这就是典型的串级控制：

- 电流环最快，直接决定瞬时转矩响应。
- 速度环次之，负责把转速误差变成转矩需求。
- 位置环最外层，负责最终位置目标。

一般调参顺序也是从里到外，先调电流环，再调速度环，最后调位置环。不然外环再漂亮，里层跟不上也没用。

### 代码实现思路

```c
// 代码实现思路
函数1：机械角度 -> 电角度，theta_e = theta_m * 极对数
函数2：把电角度归一化到 0~2pi，避免 sin/cos 输入发散
函数3：采样 ia、ib（或 ia、ib、ic），做 Clarke 变换得到 i_alpha、i_beta
函数4：根据编码器角度做 Park 变换，得到 i_d、i_q
函数5：电流环 PI，输入目标 i_d/i_q 和反馈 i_d/i_q，输出 u_d、u_q
函数6：逆 Park + 逆 Clarke，得到三相目标电压 u_a、u_b、u_c
函数7：将三相电压钳位到母线范围内，并做 SVPWM/零序注入
函数8：把相电压换算成 PWM 占空比，写入定时器比较寄存器
```
