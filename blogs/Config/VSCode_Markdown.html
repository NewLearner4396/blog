VSCode编写Markdown的相关配置

NewLearner43962022-08-18ConfigmarkdownVSCode

在VSCode用Markdown愉快地记录笔记😊

需要的插件

1. Markdown All in One

书写Markdown最基础的插件，装上它其实就能书写Markdown了。

快捷键

KeyFunction
Ctrl + BToggle bold
Ctrl + IToggle italic
Ctrl + MToggle math environment
Alt + SToggle strikethrough
Alt + CToggle task list item
Ctrl + Shift + [Toggle heading (downlevel)
Ctrl + Shift + ]Toggle heading (uplevel)
Ctrl + Shift + VToggle preview
Ctrl + K, VToggle preview to side

如果预览无法加载图片，则在VSCode编辑页面，按快捷键Ctrl + Shift + P，搜索markdown 更改预览安全设置， 选择允许不安全内容，即可预览图片啦🤒

常用命令

在VSCode命令行Ctrl + Shift + P

  • Markdown All in One: Create Table of Contents

    （在光标处创建一个目录，实际上是根据标题层次创建引用)

  • Markdown All in One: Update Table of Contents

  • Markdown All in One: Add/Update section numbers

    增加章节数字层次，如2变为2.1

  • Markdown All in One: Remove section numbers

  • Markdown All in One: Toggle list

    • It will cycle through list markers (-, *, +, 1. and 1))

支持渲染的语言

  • CommonMarkopen in new window
  • Tablesopen in new window, strikethroughopen in new window and task listsopen in new window (from GitHub Flavored Markdown)
  • Math supportopen in new window (from KaTeX)
  • Front matteropen in new window

​ 其他的Markdown语法需要安装其他的插件，如： Mermaid diagramopen in new window, emojiopen in new window, footnotesopen in new window and superscriptopen in new window

或者直接安装Markdown Preview Enhanced🤘

2. Markdown Preview Enhanced

一个支持自动同步滚动，各种拓展Markdown语法的插件

因为只是个增强插件，没什么需要配置的

快捷键

KeyFunction
Ctrl + K, VOpen preview to the side
Ctrl + shift + VOpen preview
Ctrl + Shift + SSync preview / Sync source
Ctrl + =preview zoom in
Ctrl + -preview zoom out
Ctrl + 0preview zoom reset
escToggle sidebar TOC

该插件支持直接渲染代码块运行结果🏃，快捷键为Shift + Enter

具体请查看官方文档:https://shd101wyy.github.io/markdown-preview-enhanced/#/zh-cn/code-chunkopen in new window

3. Markdown PDF

支持将Markdown导出为PDF，PNG，JPEG，HTML的插件

该插件可以自定义的部分蛮多，比如：

页边距，渲染样式，语法高亮样式等等

可以根据说明自己在设置里调整

该插件默认在右键菜单中设置了6个命令，我觉得我用不到这么多命令，而且太占空间，于是我修改了该插件的JSON文件，我的路径为：C:\Users\Administrator\.vscode\extensions\yzane.markdown-pdf-1.4.4\package.json

只留下这两个大括号就好了，记得把group的序号改一下

如果不自定义style的话，渲染出来的很朴素

使用whitey.css样式后，标题、字体等都有了变化

可以从网上下载好看的CSS样式

typora官网主题的CSS：https://theme.typora.io/open in new window

（上面那个网站打不开可以试试typora中文网：https://www.typora.net/themes/open in new window，但是主题比那个少）

注意设置的时候要输入.css文件的绝对路径哦，并且\要更改为\\

4. markdownlint

一个检查Markdown文件书写规范的插件，并且可格式化文件去除违规问题(在编辑器右键即可看到)。

该规范意使Markdown源文件简洁易读，如不喜欢可以不用。

规范文档：官方Rulesopen in new window

下面附上我个人关于Markdown在settings.JSON中的额外设置：

    "[markdown]":{
        // 避免在打字符串和注释时出现建议输入
        // 打开该设置可以使用‘markdown.json’中配置的用户代码块
            "editor.quickSuggestions": {
                "other": "on",
                "comments": "off",
                "strings": "off"
            },
        // 修改文件默认编码为utf8，因为一般网站使用的编码是这个
            "files.encoding": "utf8",
        // 修改Markdown的默认纠错器
            "editor.defaultFormatter": "DavidAnson.vscode-markdownlint",
        },

Markdown常用语法

想要分割线，可用–--、***、___表示

可以使用任务列表达到复选框的效果- [ ]

写文字少不了用各种表情调剂气氛，可用:emoji:表示，如:blush:就是😊

该网站收录了各emoji表达方式:https://www.webfx.com/tools/emoji-cheat-sheet/open in new window

还可以用Win + .调出系统的表情工具。

markdown支持写流程图，但可以用其他可视化工具代替此功能，用代码写有点不方便。
