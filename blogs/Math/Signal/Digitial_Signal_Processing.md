## 《数字信号处理》的笔记

《信号与系统》更像在回答“信号和系统本身是什么”，《数字信号处理》更关心“这些东西怎么在机器里真的做出来，而且做得快、做得稳、误差可控”。

借 [《信号与系统》的笔记](Signal_and_System.md) 里的一句话，这本书最核心的能力，就是“在时域、频域和变换域之间切换视角，最后把视角落成算法和结构”。

所以读这本书，不要只把它看成《信号与系统》的离散版。它比那更往前走一步。它关心的不只是理论对不对，还关心：

- 采样之后信息丢没丢
- 差分方程怎样实现成硬件和代码
- 频谱怎样算得足够快
- 滤波器指标怎样落成结构
- 有限字长会把理想系统扭曲到什么程度

全文的主线可以压成下面这张图：

```text
模拟信号
-> 采样
-> 离散时间信号
-> z 变换 / DTFT / DFT
-> 频率响应与系统结构
-> FIR / IIR 设计
-> FFT 加速计算
-> 谱分析、建模、解卷积
-> 回到真实实现中的量化和误差
```

如果说《信号与系统》像是在学“世界运行的规则”，那《数字信号处理》更像是在学“怎样按这些规则搭机器”。

## 第1章 绪论

绪论的任务不是介绍目录，而是先告诉你：为什么要数字化，数字化会带来哪些新问题，整本书在解决什么矛盾。

### 1.1 为什么要数字化

模拟系统直接处理连续信号，电路结构直观，但缺点也明显：

- 元件参数会漂
- 温度、老化、噪声会影响性能
- 复杂系统难修改
- 做高阶精密滤波器成本高

数字系统的优势在于：

- 同一套算法可以反复复制，参数更稳定
- 修改设计常常只需要改程序或系数
- 复杂逻辑容易分层
- 便于存储、压缩、传输和加密

从工程眼光看，数字化像是把一套容易受物理细节影响的“模拟手工活”，改造成按离散步骤执行的“程序化流程”。

### 1.2 数字化不是白来的

数字系统也不是占尽便宜。它把模拟问题搬进离散世界时，引入了三类新麻烦：

1. 采样：时间不再连续，可能混叠
2. 量化：幅值不再连续，可能引入量化噪声
3. 有限字长：乘加不是实数精确运算，可能出现溢出、舍入误差和极限环

所以 DSP 的真实风格不是“理论更高级”，而是“多了一层现实约束”。

### 1.3 这本书的工具箱

整本书在反复使用几种工具：

- `DTFT`：看离散时间信号的连续频谱
- `z` 变换：看系统函数、极点、稳定性
- `DFT`：把有限数据变成可计算的离散频谱
- `FFT`：把 DFT 算快
- `滤波器设计方法`：把需求落成系数和结构

把这几种工具关系看清，比单背定义更重要：

```text
DTFT 负责“理论频谱”
DFT 负责“有限数据的离散频谱”
FFT 负责“把 DFT 算得快”
z 变换负责“把系统分析做成代数和几何问题”
```

## 第2章 离散时间信号与系统

这一章承接《信号与系统》的前半部分，但会更偏向 DSP 后续需要的语言。

### 2.1 离散时间信号

离散时间信号写作 `x[n]`，索引 `n` 是整数。一个好习惯是把它看成“按样点编号排开的序列”，而不只是一个下标函数。

常见基本序列：

单位冲激：

```math
\delta[n]=
\begin{cases}
1, & n=0 \\
0, & n\ne 0
\end{cases}
```

单位阶跃：

```math
u[n]=
\begin{cases}
1, & n\ge 0 \\
0, & n<0
\end{cases}
```

复指数：

```math
x[n]=Ae^{j\omega_0 n}
```

正弦：

```math
x[n]=A\cos(\omega_0 n+\phi)
```

#### 为什么这些基本序列重要

因为它们像积木。

- 冲激适合做分解
- 阶跃适合表示启动和截断
- 复指数适合做频域分析
- 正弦适合和实际振荡信号对应

#### 周期性条件

离散时间正弦或复指数要周期，必须存在正整数 `N` 使

```math
\omega_0 N = 2\pi k
```

所以

```math
\frac{\omega_0}{2\pi}
```

必须是有理数。

直觉上，这是因为每走一步就绕单位圆转一个固定角度，只有这个角度和整圆能整除，绕若干步后才能回到原点。

### 2.2 离散时间系统

系统写作

```math
y[n]=T\{x[n]\}
```

最常见的分类方式有：

- 线性 / 非线性
- 时不变 / 时变
- 因果 / 非因果
- 稳定 / 不稳定
- 递归 / 非递归

#### 递归与非递归

这是 DSP 里很重要的一组分类。

- 非递归系统：当前输出只依赖输入，不依赖过去输出
- 递归系统：当前输出还依赖过去输出

递归系统有记忆，往往更省阶数，但也更容易受量化和稳定性影响。

### 2.3 LTI 系统

LTI 系统仍然是主角，因为它既足够常见，又足够好算。

任意序列都可写成冲激展开：

```math
x[n]=\sum_{k=-\infty}^{\infty}x[k]\delta[n-k]
```

设冲激响应为 `h[n]`，则输出：

```math
y[n]=\sum_{k=-\infty}^{\infty}x[k]h[n-k]
```

也就是

```math
y[n]=x[n]*h[n]
```

卷积在 DSP 里的地位极高。你看到“数字滤波器”的时候，很多时候可以直接翻译成“一个卷积或带反馈的卷积结构”。

#### 冲激响应像什么

可以把冲激响应理解成系统的“回声模板”。

输入若在某个样点打入一个单位冲激，系统之后怎样回响，这个模板就是 `h[n]`。任意输入都能看成许多冲激的叠加，所以任意输出也就是许多回响的叠加。

### 2.4 线性常系数差分方程

大量离散系统都可写成：

```math
\sum_{k=0}^{N}a_k y[n-k]
=
\sum_{m=0}^{M}b_m x[n-m]
```

看这个方程时，不妨分左右两边读：

- 右边：输入如何推动系统
- 左边：系统内部记忆如何把过去输出带过来

若只含输入延时项，系统往往是 FIR。

若含过去输出项，系统往往是 IIR。

### 2.5 频域表示

DTFT 定义为

```math
X(e^{j\omega})=\sum_{n=-\infty}^{\infty}x[n]e^{-j\omega n}
```

逆变换：

```math
x[n]=\frac{1}{2\pi}\int_{2\pi}X(e^{j\omega})e^{j\omega n}d\omega
```

#### 为什么频域有用

因为卷积太烦，频域会把它变乘法：

```math
Y(e^{j\omega})=X(e^{j\omega})H(e^{j\omega})
```

这和《信号与系统》里是同一件事，但在 DSP 里它更直接地意味着：

```text
做滤波
=
让不同频率乘上不同权重
```

这句朴素的话，后面会贯穿整本书。

### 2.6 离散时间随机信号

这部分在绪论里往往不抢眼，但后面做谱估计、建模和噪声分析时都要靠它。

随机信号不是“每次都不同所以没法研究”，而是研究它的统计规律，比如均值、自相关、功率谱密度。

一个常见直觉是：

- 确定性信号问“这一时刻等于多少”
- 随机信号问“这一时刻大概长什么分布，与别的时刻相关不相关”

## 第3章 z 变换

这一章是 DSP 理论工具箱里的中轴。

DTFT 很适合看频谱，但不擅长处理收敛域、因果性和差分方程。z 变换把这些事情接到一起。

### 3.1 z 变换定义

```math
X(z)=\sum_{n=-\infty}^{\infty}x[n]z^{-n}
```

若写成

```math
z=re^{j\omega}
```

则

```math
z^{-n}=r^{-n}e^{-j\omega n}
```

你可以看到，它比 DTFT 多了一个径向因子 `r^{-n}`。这相当于给序列又乘了一层指数权重，让原本可能发散的和式在某些区域变得收敛。

### 3.2 收敛域 ROC

ROC 是 z 变换里最容易被忽视、但实际最关键的部分。

同一个代数式，没有 ROC，时域信号可能不唯一。

对于有理 z 变换，常见规律如下：

- 右边序列的 ROC 在最外极点之外
- 左边序列的 ROC 在最内极点之内
- 双边序列的 ROC 在两个极点圆之间
- 稳定要求 ROC 包含单位圆
- 因果稳定有理系统要求所有极点都在单位圆内

#### 为什么稳定看单位圆

因为频率响应来自

```math
H(e^{j\omega})=H(z)\big|_{z=e^{j\omega}}
```

也就是说，做频域分析时你正沿着单位圆跑一圈。若单位圆不在 ROC 里，这一圈就不合法，频率响应都没法完整定义。

### 3.3 z 逆变换

常见方法：

1. 查表
2. 部分分式展开
3. 幂级数展开
4. 复积分

最常用的是部分分式展开，因为它能把复杂有理函数拆成若干简单极点项，而每个简单项往往对应基本指数序列。

例如

```math
\frac{1}{1-az^{-1}}
```

常对应

```math
a^n u[n]
```

或

```math
-a^n u[-n-1]
```

真正决定是哪一个的，是 ROC。

### 3.4 z 变换性质

线性、时移、卷积这些都和傅里叶变换相通。

例如卷积性质：

```math
x[n]*h[n]\leftrightarrow X(z)H(z)
```

这条式子把“系统输出是卷积”直接翻译成“系统输出的 z 变换是乘法”，为系统函数的定义铺路。

### 3.5 z 变换与 LTI 系统

系统函数定义为

```math
H(z)=\frac{Y(z)}{X(z)}
```

若系统满足差分方程

```math
\sum_{k=0}^{N}a_k y[n-k]
=
\sum_{m=0}^{M}b_m x[n-m]
```

则

```math
H(z)=
\frac{\sum_{m=0}^{M}b_m z^{-m}}
{\sum_{k=0}^{N}a_k z^{-k}}
```

一旦把它写成 `H(z)`，很多问题就清楚了：

- 看分母零点，也就是极点，判断稳定和记忆
- 看分子零点，判断哪些频率会被压制
- 在单位圆上取值，得到频率响应

### 3.6 单边 z 变换

单边 z 变换适合处理有初始条件的递推系统启动过程。它和双边 z 变换不是互相替代，而是处理场景不同：

- 双边 z 变换适合一般系统分析
- 单边 z 变换适合解带初始状态的差分方程

## 第4章 连续时间信号的采样

这一章是数字处理真正和现实世界接触的地方。

如果没有采样，离散时间处理就无从谈起；如果采样做得不好，后面再漂亮的算法都补不回来。

### 4.1 周期采样

理想采样可表示为

```math
x_s(t)=x_a(t)\sum_{n=-\infty}^{\infty}\delta(t-nT_s)
```

其中 `x_a(t)` 是模拟信号，`T_s` 是采样周期。

这个式子看起来抽象，实际意思很简单：用一串等间隔冲激去“戳”连续信号，每个戳点留下一个样值。

### 4.2 采样的频域表示

采样后频谱会周期复制：

```math
X_s(j\omega)=\frac{1}{T_s}\sum_{k=-\infty}^{\infty}
X_a\left(j(\omega-k\omega_s)\right)
```

其中

```math
\omega_s=\frac{2\pi}{T_s}
```

这条式子值得慢读。它告诉你：

- 采样不是“只在时间上变稀”
- 它在频域的代价是“原谱被一份份平移复制”

### 4.3 由样本重构带限信号

如果原信号带限到 `|\omega|<\omega_m`，且满足

```math
\omega_s > 2\omega_m
```

那么复制出来的频谱彼此不重叠，就能用理想低通滤波取回原谱。

重构公式：

```math
x_a(t)=\sum_{n=-\infty}^{\infty}x[n]
\mathrm{sinc}\left(\frac{t-nT_s}{T_s}\right)
```

#### sinc 插值的直觉

每个样点不只是一个孤立数字，而像是一条中心在该样点的 sinc 小波形。所有样点对应的 sinc 波形叠起来，刚好把原曲线补全。

### 4.4 连续时间信号的离散时间处理

真实数字系统常写成

```text
模拟输入 -> A/D -> 数字处理 -> D/A -> 模拟输出
```

这一结构的本质是：你先把连续信号搬进离散世界，利用数字系统的灵活性完成处理，再搬回连续世界。

### 4.5 改变采样率

抽取 `M` 倍：

```math
y[n]=x[Mn]
```

它会压缩频谱周期，若不先低通就会混叠。

插值 `L` 倍通常分两步：

1. 插零
2. 低通滤波

插零后频谱会出现镜像，需要低通把多余副本切掉。

### 4.6 多采样率信号处理

多采样率系统的核心不是多一个公式，而是多一种设计策略：

```text
在不同采样率下做不同运算，
以换取更低复杂度或更合适的频谱分布。
```

这在通信、语音、图像和音频处理中都很常见。

### 4.7 过采样和噪声形成

量化噪声的总能量很难凭空消失，但你可以改变它在频域里的分布。

- 过采样：把噪声摊到更宽频带
- 噪声形成：把噪声推向高频

这样在目标低频带内，实际保留下来的噪声会更小。

## 第5章 线性时不变系统的变换分析

这一章在问：当你已经知道一个系统的 `H(z)` 或 `H(e^{j\omega})` 后，如何进一步读懂它的幅度、相位、最小相位和线性相位性质。

### 5.1 LTI 系统的频率响应

若输入

```math
x[n]=e^{j\omega n}
```

输出为

```math
y[n]=H(e^{j\omega})e^{j\omega n}
```

这说明每个频率分量在 LTI 系统中彼此独立地被缩放和旋转。

所以复杂波形经过系统后变成什么，不需要盯着整条曲线看，只要看每个频率的系数怎样变化。

### 5.2 有理系统函数的频率响应

若

```math
H(z)=K\frac{\prod_i(1-c_i z^{-1})}{\prod_k(1-d_k z^{-1})}
```

则在单位圆上

```math
H(e^{j\omega})=
K\frac{\prod_i(1-c_i e^{-j\omega})}{\prod_k(1-d_k e^{-j\omega})}
```

可以做几何理解：

- 分子来自到零点的距离
- 分母来自到极点的距离

因此：

- 极点靠近单位圆某处，会在附近把响应抬高
- 零点靠近单位圆某处，会在附近把响应压低

### 5.3 幅度和相位之间的关系

很多初学者把幅度响应和相位响应看成两张独立图。其实在某些系统类里，它们并不完全独立。

尤其是最小相位系统，幅度特性会强约束相位特性。

直觉上，最小相位系统在实现同样幅频响应时，倾向于把能量和响应尽量往前集中，不把不必要的延迟拖到后面。

### 5.4 全通系统

全通系统满足

```math
|H(e^{j\omega})|=1
```

也就是说，它对所有频率幅度一视同仁，不做衰减和放大。

但相位可以很复杂。所以全通系统的作用不是改频谱高低，而是改时间排列关系和波形相位结构。

### 5.5 最小相位系统

离散时间最小相位系统常指：

- 因果
- 稳定
- 逆系统也因果稳定

对于有理系统，这通常意味着零点和极点都在单位圆内。

“最小相位”这个名字容易让人误以为相位最小值最小。更准确的理解是：在实现同样幅度响应的系统里，它的相位滞后和时域拖尾相对更小。

### 5.6 广义线性相位系统

FIR 系统若满足系数对称或反对称：

```math
h[n]=h[N-1-n]
```

或

```math
h[n]=-h[N-1-n]
```

则常能得到线性相位：

```math
\angle H(e^{j\omega})=-\omega \alpha + \phi_0
```

线性相位的重要性在于，它让所有频率分量延迟一致，从而较好保留波形形状。

## 第6章 离散时间系统结构

前面讲的是“系统是什么”，这一章讲“系统怎么搭”。

同一个传递函数，可能对应多种实现结构。理论上它们等价，数值效果和硬件代价却可能很不一样。

### 6.1 差分方程的方框图表示

例如

```math
y[n]=b_0x[n]+b_1x[n-1]-a_1y[n-1]
```

它通常会画成：

```text
x -->(*)--+
          |
x -> z^-1 ->(*)--+--> sum --> y
                 ^
                 |
y -> z^-1 ->(*)--+
```

这里

- `z^{-1}` 表示一拍延时
- `(*)` 表示乘系数
- `sum` 表示求和

方框图的价值在于，它把代数方程变成“数据流如何流动”的结构图。

### 6.2 信号流图

信号流图比方框图更强调节点和支路的关系。它适合分析复杂系统互联和推导等效结构。

### 6.3 IIR 系统的基本结构

常见结构：

- 直接型 I
- 直接型 II
- 级联型
- 并联型
- 转置型

#### 如何理解这些结构差异

不是“换一种画法”那么简单。

- 直接型 I 结构直观
- 直接型 II 更省存储
- 级联型适合按二阶节实现，数值上常更稳
- 并联型适合部分分式拆解后的实现
- 转置型在硬件和数值特性上有时更方便

### 6.4 FIR 系统结构

FIR 没有反馈，天然稳定。若系数对称，还能通过共用乘法减少运算量。

这也是为什么很多要求线性相位和高稳健性的场景会偏爱 FIR。

### 6.5 格型滤波器

格型结构常出现在预测、自适应和语音建模中。它用一组反射系数逐层构造系统，数值性质通常更好，也更利于递推更新。

### 6.6 有限精度效应

这一节非常工程，但不能略过。

在机器里：

- 系数要量化
- 乘积要截断或舍入
- 中间结果可能溢出

这些都会让“纸面上的 H(z)”和“机器里的 H(z)”不再完全一样。

常用的量化噪声模型是：

若量化步长为 `\Delta`，则均匀量化误差近似方差

```math
\sigma_q^2=\frac{\Delta^2}{12}
```

这不是铁律，但作为一阶近似很好用。

#### 极限环

IIR 系统特别可能出现零输入极限环。也就是说，即使输入已经是零，系统内部由于量化和反馈，还可能自己维持一个小振荡而不消失。

这件事很像一个本来该停下来的摆，因为每次取整误差又被反馈打回去，结果在一个很小的范围里来回晃。

## 第7章 滤波器设计方法

这一章是 DSP 最接近“需求到产品”的地方。

前面说的是分析，现在开始做设计：给定通带、阻带、波纹和过渡带要求，怎样得到一组可实现的系数。

### 7.1 技术指标

以低通滤波器为例，常见指标有：

- 通带边缘频率 `\omega_p`
- 阻带边缘频率 `\omega_s`
- 通带最大波纹 `\delta_p`
- 阻带最大波纹 `\delta_s`

图形上：

```text
|H(e^jw)|
1.0  _________
    /         \
    |  通带    \
    |           \   阻带
0.0 +------------\_______
      0   wp  ws      pi
```

设计滤波器，本质是在这些约束之间找一条能实现的曲线。

### 7.2 从连续时间滤波器设计数字 IIR

经典套路：

1. 先设计模拟原型
2. 再映射到数字域

两种主要映射法：

- 冲激响应不变法
- 双线性变换法

#### 冲激响应不变法

思路是让数字系统的冲激响应保留模拟系统冲激响应的采样值。

优点：时域直观。

缺点：频谱会折叠，可能产生混叠。

#### 双线性变换法

映射公式：

```math
s=\frac{2}{T}\frac{1-z^{-1}}{1+z^{-1}}
```

它把模拟 `j\Omega` 轴一一映射到数字单位圆上，不会产生混叠。

缺点是频率发生非线性扭曲，所以要预失真：

```math
\Omega=\frac{2}{T}\tan\left(\frac{\omega}{2}\right)
```

### 7.3 巴特沃思、切比雪夫和椭圆滤波器

这三类 IIR 原型的差别，最好不要只背名字，要记住它们是在不同折中点上站队。

巴特沃思：

- 通带单调平滑
- 没有波纹
- 滚降不算最陡

切比雪夫：

- 通带有波纹
- 换来更陡过渡带

椭圆滤波器：

- 通带、阻带都有波纹
- 在同阶数下过渡带最陡

一句话说：

```text
越想切得陡，越得接受更多波纹或更复杂相位。
```

### 7.4 FIR 设计的窗函数法

基本思路：

1. 先写理想滤波器的无限长冲激响应 `h_d[n]`
2. 再乘有限长窗 `w[n]`

```math
h[n]=h_d[n]w[n]
```

理想低通的冲激响应常写成：

```math
h_d[n]=\frac{\sin \omega_c(n-\alpha)}{\pi(n-\alpha)}
```

其中

```math
\alpha=\frac{N-1}{2}
```

#### 窗函数为什么必要

理想低通对应无限长 sinc，机器装不下。乘窗就是把它截断。

但时域一截断，频域就不再理想，所以窗函数的选择，本质上是在控制：

- 主瓣宽度
- 旁瓣高度

矩形窗主瓣最窄，但旁瓣高，泄漏重。

海明窗、汉宁窗旁瓣低一些，过渡带宽一些。

Kaiser 窗可调，是工程里很实用的一类。

### 7.5 FIR 最佳逼近与等波纹设计

Parks-McClellan 设计追求的不是“平均误差最小”，而是“最坏处误差尽量均匀压平”。

这是一种切比雪夫意义下的最优逼近。它的特征是误差曲线在若干频点之间交替达到等大极值，所以常叫等波纹设计。

### 7.6 IIR 和 FIR 的比较

IIR：

- 阶数低
- 实现省
- 相位常不线性
- 对量化更敏感

FIR：

- 一定稳定
- 易实现线性相位
- 阶数常较高
- 运算量通常更大

这不是谁绝对更好，而是看需求。若特别在意相位和稳健性，FIR 很有优势；若特别在意阶数和实时复杂度，IIR 常更划算。

## 第8章 离散傅里叶变换

DTFT 频域连续，理论上很漂亮，但计算机不可能把一整条连续函数全部保存和运算。DFT 就是在有限数据和有限频点下对频谱做离散化。

### 8.1 周期序列与离散傅里叶级数

离散时间周期序列可写成离散傅里叶级数。这一节的重要意义是把后面的 DFT 放到“周期序列频谱表示”里理解，而不是把 DFT 当成凭空出现的公式。

### 8.2 DFT 的定义

对长度为 `N` 的序列：

```math
X[k]=\sum_{n=0}^{N-1}x[n]e^{-j\frac{2\pi}{N}kn},
\quad k=0,1,\dots,N-1
```

逆变换：

```math
x[n]=\frac{1}{N}\sum_{k=0}^{N-1}X[k]e^{j\frac{2\pi}{N}kn}
```

#### DFT 到底在做什么

最实用的理解有两层：

1. 它把长度 `N` 的数据段当成一个周期为 `N` 的序列的一段
2. 它在 `N` 个等间隔频率点上采样 DTFT

所以 DFT 不是简单“求频谱”，而是带着“周期延拓”和“离散取样”这两层前提的频谱表示。

### 8.3 DFT 的性质

线性、循环移位、共轭对称等性质都很重要。

实信号的 DFT 常满足共轭对称，这意味着正频和负频并不独立。

### 8.4 线性卷积与循环卷积

DFT 域乘法对应的是循环卷积：

```math
x[n]\circledast_N h[n]
\leftrightarrow
X[k]H[k]
```

这和普通线性卷积不是一回事。

循环卷积可以理解成“序列尾巴绕回头部再相加”。如果不希望绕回造成污染，就必须补零。

若 `x[n]` 长度为 `L_x`，`h[n]` 长度为 `L_h`，要得到线性卷积，需取

```math
N \ge L_x + L_h - 1
```

### 8.5 离散余弦变换 DCT

DCT 可以看成在某些对称延拓条件下的频谱表示。它常把信号能量更集中到低频系数里，因此在压缩中很重要，比如图像块压缩。

## 第9章 离散傅里叶变换的计算

这一章主角是 FFT。DFT 理论上很好，但若直接算，复杂度太高；FFT 则把计算结构挖开，充分利用旋转因子的对称性。

### 9.1 直接计算的代价

长度 `N` 的 DFT 若逐项直接算，复杂度约为

```math
O(N^2)
```

这在大规模数据上会很慢。

### 9.2 时间抽取 FFT

核心思路是把序列拆成偶下标和奇下标两组：

```math
X[k]=\sum_r x[2r]W_N^{2rk}
     +W_N^k\sum_r x[2r+1]W_N^{2rk}
```

其中

```math
W_N=e^{-j2\pi/N}
```

于是

```math
X[k]=E[k]+W_N^k O[k]
```

```math
X[k+N/2]=E[k]-W_N^k O[k]
```

这就是蝶形结构。

#### 蝶形为什么快

因为原本很多重复乘法被共享了。FFT 真正的价值，不是发现了新结果，而是发现了“同一个结果能否少做很多重复劳动”。

### 9.3 频率抽取 FFT

频率抽取和时间抽取本质类似，只是拆分视角不同。一个更像先切输入，一个更像先切输出。

### 9.4 实现问题

FFT 实现时会遇到：

- 位反转排序
- 原地计算
- 蝶形级联
- 旋转因子存储

这些细节在纸上不显眼，写代码或硬件时却很关键。

### 9.5 更一般的 FFT

除了 `2` 的幂次长度，还有混合基、素因子、Bluestein 等更一般算法。核心精神不变：利用结构减少重复计算。

### 9.6 有限寄存器长度影响

FFT 不只是快，还得抗误差。

在多级蝶形中：

- 系数量化会传播
- 舍入误差会累积
- 中间节点可能溢出

所以 FFT 设计也常要配合缩放策略和字长规划。

## 第10章 利用 DFT 的信号傅里叶分析

这一章讲的是“拿一段数据做频谱分析时，实际会发生什么”。

它非常实战，因为真实世界里你拿到的从来不是无限长信号，而是一段有限记录。

### 10.1 用 DFT 分析信号

当你对有限长数据做 DFT 时，实际上等于：

```text
原信号
-> 乘一个有限长度窗
-> 再做频谱分析
```

而时域乘窗会在频域变卷积，这就解释了为什么“看似简单的做个 DFT”会出现各种谱泄漏和分辨率问题。

### 10.2 正弦信号的 DFT 分析

若正弦频率正好落在 DFT 栅格点上，谱会集中在对应频点。

若不正好对齐，能量会洒到周围频点，这叫谱泄漏。

#### 栅栏效应

DFT 只在有限个频点上取样，像透过一排栏杆看风景。若目标正落在栏杆缝中央，看得很准；若夹在两个栏杆之间，就会显得分散。

### 10.3 窗函数和谱泄漏

窗函数改变的是泄漏分布：

- 矩形窗主瓣窄，旁瓣高
- 更平滑的窗主瓣宽，旁瓣低

于是你总是在“频率分辨率”和“旁瓣泄漏”之间做权衡。

### 10.4 短时傅里叶变换

非平稳信号频谱会随时间变。于是要分帧：

```math
X_m[k]=\sum_n x[n]w[n-mR]e^{-j2\pi kn/N}
```

这里窗口沿时间滑动，于是你得到的是“时间-频率图像”。

短时傅里叶变换像是拿一把移动的小窗，一段一段地听信号在每个局部的频率成分。

### 10.5 平稳随机信号的谱分析

周期图是基本功率谱估计：

```math
P_{xx}(e^{j\omega}) \approx \frac{1}{N}|X_N(e^{j\omega})|^2
```

但它方差大，所以实际中常用分段平均、加窗平均等方法降低估计波动。

## 第11章 参数信号建模

前面主要是在“直接看频谱”。这一章开始换思路：

```text
假设信号是由某个模型生成的，
那我能不能通过观测数据把模型参数倒推出来？
```

### 11.1 全极点建模

经典 AR 模型：

```math
x[n]+\sum_{k=1}^{p}a_k x[n-k]=e[n]
```

这里 `e[n]` 常看成白噪声激励。

对应系统函数：

```math
H(z)=\frac{1}{1+\sum_{k=1}^{p}a_k z^{-k}}
```

之所以叫全极点模型，是因为分子通常是常数，没有显式零点结构。

#### 直觉

这类模型假设：当前样值大体可以由过去若干样值线性预测，剩下那点预测不了的残差当成白噪声。

### 11.2 确定性和随机信号建模

建模不是只有一种风格。

- 对确定性周期分量，常看频率、幅值、相位
- 对随机过程，常看自相关、功率谱和参数模型

### 11.3 相关函数估计

理论自相关：

```math
r_{xx}[m]=E\{x[n]x^*[n-m]\}
```

有限数据下通常只能估计：

```math
\hat r_{xx}[m]=\frac{1}{N}\sum_n x[n]x^*[n-m]
```

自相关反映的是“一个信号和自己延迟版本的相似度”。若某个延迟下相似度很高，说明信号中可能存在对应周期或记忆结构。

### 11.4 模型阶数

阶数选得太低，模型太粗；选得太高，可能把噪声也拟合进去。

AIC、MDL 之类准则都是在平衡：

- 拟合能力
- 模型复杂度

### 11.5 全极点谱分析

参数谱分析的优势是：哪怕数据短，若模型假设合适，也可能得到比直接 DFT 更平滑、更高分辨率的谱估计。

代价是：模型若选错，结果可能看起来很漂亮，但解释其实错了。

### 11.6 正规方程与 Levinson-Durbin

AR 参数常由 Yule-Walker 方程给出：

```math
\sum_{k=1}^{p}a_k r[m-k]=-r[m], \quad m=1,\dots,p
```

因为相关矩阵是 Toeplitz 结构，可用 Levinson-Durbin 递推高效求解。

### 11.7 格型滤波器

格型结构在这里再次出现，不是巧合。它和线性预测的反射系数之间关系很紧，既方便递推求解，也有良好的数值性质。

## 第12章 离散希尔伯特变换

这一章处理的是一个很有意思的问题：实信号的频谱正负两边不是独立的，那能不能构造一个和原信号成 `90` 度相移的配对信号？

### 12.1 希尔伯特变换的作用

希尔伯特变换本质上是在频域对正频和负频施加不同相位移：

- 正频率移 `-\pi/2`
- 负频率移 `+\pi/2`

这样得到的新信号和原信号正交。

### 12.2 解析信号

通过原信号与其希尔伯特变换构造：

```math
x_a[n]=x[n]+j\hat x[n]
```

解析信号的妙处是，它只保留正频率分量。

#### 为什么有用

因为许多调制、包络、瞬时频率问题，用单边频谱表达会更干净。原来正负频缠在一起的东西，可以分开处理。

### 12.3 幅度与相位关系

最小相位系统里，对数幅度和相位之间存在希尔伯特变换关系。这说明某些系统中，幅频和相频其实深度耦合，不是你想怎么配就怎么配。

## 第13章 倒谱分析和同态解卷积

这一章第一次看常让人发懵，因为它把频域再做了一次对数和逆变换。但思路其实很干净。

卷积在频域里会变乘法，而乘法取对数就会变加法。加法比卷积好分离，所以倒谱是为了“把卷在一起的东西拆开”。

### 13.1 倒谱定义

实倒谱常写成：

```math
c[n]=\mathcal{F}^{-1}\{\log |X(e^{j\omega})|\}
```

复倒谱则进一步保留相位信息。

### 13.2 为什么倒谱能做解卷积

若

```math
y[n]=x[n]*h[n]
```

则频域上

```math
Y=XH
```

取对数：

```math
\log Y = \log X + \log H
```

再逆变换后，原本卷在一起的东西在倒谱域有机会分离。

一句话说：

```text
倒谱不是另一种频谱，
而是为了把乘法关系拆成加法关系。
```

### 13.3 多径和语音中的应用

语音常可粗略看成：

```text
激励源 * 声道系统
```

在倒谱域里，激励周期信息和声道包络信息常出现在不同 quefrency 范围，于是可以分开看。

多径问题里也是类似：回波导致的卷积结构，在倒谱域里往往更容易露出分离线索。

### 13.4 最小相位、最大相位与复倒谱

复倒谱和零点分布关系很紧。最小相位序列、最大相位序列的分解与重构，经常要靠这套语言来讲清楚。

## 最后收一下

把《数字信号处理》压成一条主线，就是：

1. 连续信号先经采样进入离散世界。
2. 离散世界里，LTI 系统仍然是主角，差分方程和 z 变换负责描述它。
3. 频域里用 DTFT 看理论频谱，用 DFT 看有限数据频谱，用 FFT 加速计算。
4. 设计层面上，IIR 和 FIR 给出两种主要滤波器路线。
5. 实现层面上，还必须考虑结构、量化、舍入和有限字长。
6. 更高层的谱分析、参数建模、希尔伯特变换和倒谱，则是在这些基本工具上继续往前走。

```text
《数字信号处理》真正难也真正有意思的地方，不在于公式本身，而在于你得同时看懂理论、结构、算法和误差，少看一层都不完整。
```
